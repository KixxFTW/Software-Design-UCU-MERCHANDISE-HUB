Security Features

  • Secure password hashing using bcrypt
    -Turns passwords into secret codes so they can't be stolen easily.

  • Session-based authentication
    -Ensures that only authorized users can access certain parts of the system.

  • CSRF protection
    -Prevents cross-site request forgery attacks, which can compromise user data.

  • Input sanitization
    -Prevents malicious input from being processed by the system.

  • Role-based access control
    - Only lets users do things they’re allowed to — like admins vs. regular users.

Database Tables Overview

👤 User Management

  • STUDENTS - Student accounts (local + Google auth)
  • EDUCATORS - Instructor accounts (Google auth)
  • NOTIFICATION_PREFERENCES - Student notification settings
  • EMAIL_OTP - OTP codes for verification
  • PASSWORD_RESETS - Password reset tokens

🛒 E-commerce

  • MERCHANDISE - Products with category & stock
  • CART_ITEMS - Shopping cart line items

💳 Order & Payment

  • ORDERS - Orders with delivery & payment status
  • ORDER_ITEMS - Items in each order
  • PAYMENTS - Payment records by reference number

Relationship Key Definitions

Relationship Type Description Example
One-to-Many (1..M) One entity can have multiple related entities One Student can place many Orders
Many-to-One (M..1) Many entities relate to one entity Many Order_Items belong to one Order
One-to-One (1..1) One entity relates to exactly one other One Student has one Notification Preferences row
